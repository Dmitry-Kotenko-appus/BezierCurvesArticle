Bezier Curves in App Development

Introduction

Modern application development can’t be imagined without animations or smooth moves. But we can not write movement logic for every time of animation. However there is strong and simply math mechanism for it , called Bezier Curves.

Definition

A Bézier curve is a parametric curve frequently used in computer graphics and related fields. Generalizations of Bézier curves to higherdimensions are called Bézier surfaces, of which the Bézier triangle is a special case.

The points Pi are called control points for the Bézier curve. The polygon formed by connecting the Bézier points with lines, starting with P0 and finishing with Pn, is called the Bézier polygon(or control polygon). The convex hull of the Bézier polygon contains the Bézier curve.

We can use bezier curves for animating of views movement. The advantage of using curves is that we can create non-linear movement

                          

 IOS functionality for curves:

Draw Curve:

Apple provides native functionality for drawing of curves.

UIBezierPath (https://developer.apple.com/library/ios/documentation/UIKit/Reference/UIBezierPath_class/)

A UIBezierPath object combines the geometry of a path with attributes that describe the path during rendering. You set the geometry and attributes separately and can change them independent of one another. Once you have the object configured the way you want it, you can tell it to draw it self in the current context.

Animation:

                          

 One of the powerful mechanisms of curves usage is animation.

You can create your own animation for object using CGPath in few simple steps.

CGPath is a structure that contains description of your curve and can be simply created of fetched from UIBezierPath object.

CGPath animation path = path.CGPath;

There are a lot of instruments for animating by curves path , but in our example will be used CAKeyframeAnimation.

We can set for this object movement path and add it to animation list of layer.

CAKeyframeAnimation *move = [CAKeyframeAnimation animationWithKeyPath:@"position"];

move.path = path.CGPath;

move.duration = 10.0f;

move.repeatCount = HUGE_VALF;

self.pencil.layer.anchorPoint = CGPointMake(0.f, 1.f);

self.pencil.layer.position = path.currentPoint; // instead of removeOnCompletion

[self.pencil.layer addAnimation:move forKey:@"move the view"];

This method of animation is also good for custom loading indicator where you know path for movement of loading indicator and need to change during the progress filled part , speed of animation and etc.
